P0 · 本周内完成（直接影响 Google 流量）

P0 · #1约 20 分钟

首页加 H1/H2 —— 现在整页没有一个大标题标签

为什么：Hero 大标题「Best Sellers」实际是两个 h5 标签（其中一个是空的），Google 读不懂页面主题，自然流量入口等于焊死。这是全部问题里对流量影响最大的一条。
  1. 后台左侧 页面 → 找到首页 → 编辑（进入块编辑器）
  2. 点中轮播图里的大标题「Best Sellers」文字块（Stackable 标题块），右侧设置面板找到 HTML 标签 / Heading Tag 下拉（h1~h6），把 h5 改成 h1
  3. 标题文案顺手改成含关键词的，建议：Personalized Baby Shower Gifts & Name Baskets（第二张轮播「Baby Shower」同法处理，但整页只能有一个 h1——第二张设为 h2）
  4. 注意：实测有一个空的 h5 标题块（没有文字），找到后直接删除
  5. 往下滚，把「A GIFT AS UNIQUE AS THEIR NAME」等每个分区标题设为 h2
  6. 右上角 更新 保存
验证：打开 namelicraft.com 首页 → 右键「查看网页源代码」→ Ctrl+F 搜 <h1 → 能搜到且只有 1 个 = 成功。
P0 · #2约 10 分钟

重写首页 Title —— 现在一个产品关键词都没有

为什么：当前 title 是 Nameli | Personalized Gifts，baby / custom / embroidered / basket 全没进——搜索个性化婴儿礼品的人 Google 根本不知道该给你排名。
  1. 编辑首页时，页面下方找到 Yoast SEO 面板
  2. SEO 标题改为：Personalized Baby Shower Gifts & Custom Name Baskets | Nameli（约 60 字符内）
  3. 元描述（Meta description）补一句 150 字符内的卖点，建议：Hand-embroidered name baskets, backpacks & baby gifts. Spelling double-checked on every order. Free US shipping over $75.
  4. 产品页同理：如 Baby Shower Basket 的 title 建议加上 Personalized → Personalized Baby Shower Basket with Name | Nameli
验证：首页源代码搜 <title>，确认新标题已输出（缓存插件记得先清）。
P0 · #3每个产品约 30 分钟 · 先做 Top3 SKU

产品页补文字卖点 + 全站图片补 alt

为什么：实测主力产品 Baby Shower Basket 文字描述只有 184 字符，工艺/规格/故事全在 6 张长图里——人眼看着丰富，但 Google、AI 搜索、读屏器一个字都读不到；首页 39 张图有 27 张没有 alt。这是 SEO 命脉级修复。
  1. 后台 产品 → 编辑 Baby Shower Basket（先做卖得最好的 3 个 SKU）
  2. 在产品描述区，每张卖点长图下方补一段对应的文字版（合计 200-300 词）：材质（cotton rope）、尺寸（L/M 具体 cm）、刺绣工艺（hand-embroidered wool yarn name）、礼盒清单（内含哪些件）、发货时效
  3. 不要删图——图照留，文字是补给搜索引擎和读屏器的
  4. 图片 alt：后台 媒体 → 点开每张产品图 → 右侧替代文本填一句描述，例：Personalized baby shower basket with hand-embroidered name Oliva（先补首页和 Top3 产品页用到的图）
验证：产品页源代码搜补写的那段文字能搜到；随机点 3 张图源代码看 alt="..." 非空。
P0 · #4约 15 分钟

移动端悬浮件遮挡 —— VIP OFFER 浮标盖住了卖点文字

为什么：手机上打开产品页，左下角 VIP OFFER 圆形浮标直接压住产品副标题和卖点列表（实测截图见诊断页 #3）。手机是主要流量入口，用户看卖点的黄金位置被自家挂件占了。
  1. 这个浮标是 Klaviyo 弹窗表单关闭后留下的 Teaser 按钮（判断依据：它只在弹过窗的会话中出现，与弹窗同生命周期）
  2. 登录 Klaviyo → Sign-up forms → 找到「Get a 10% off coupon」这个表单 → 编辑
  3. 找到 Teaser 选项卡：关闭 Mobile 端 Teaser（推荐），或把位置从左下改到不压内容的位置
  4. 同一个编辑器里顺手做 P2 · #8：Display / Targeting & Behavior 里把弹出时机从「立即」改成 滚动 50% 或停留 15 秒——广告落地流量第一眼别吃全屏弹窗
验证：手机开无痕窗口访问产品页 → 关掉弹窗后左下角不再有浮标压住文字。

P1 · 两周内完成（信任感细节）

P1 · #5约 10 分钟

「Write a Review」按钮白字看不清（对比度 1.71，标准要 4.5）

  1. 优先在 WiserReview 后台找按钮颜色设置：文字色改深棕 #5C4327（同色系不破坏奶油调性）或底色加深
  2. 如果 WiserReview 没有颜色选项，用 CSS 兜底：后台 外观 → 自定义 → 附加 CSS，粘贴：
/* Write a Review 按钮文字加深（原白字配奶茶底看不清） */
.Wsrspfapp-reviewapp-heAder { color: #5C4327 !important; }
验证：首页评价区的「Write a Review」肉眼清晰可读。
选择器 .Wsrspfapp-reviewapp-heAder 为 2026-07-12 实测抓取；若 WiserReview 更新后失效，右键该按钮→检查，换成当时的 class。
P1 · #6约 10 分钟

信任条「208 reviews on ___」断句 —— logo 显示宽度为 0

  1. 方案 A（推荐，更干净）：在 WiserReview 挂件设置里把这条徽章文案换成自有表述 4.8/5 · 208 verified reviews 加星标，不依赖第三方 logo
  2. 方案 B（保留 logo）：附加 CSS 强制给 logo 宽度：
/* 信任条 WiserReview logo 渲染宽度为 0 的修复 */
.Wisely-Fixed-Typone-Link img { width: 90px !important; height: auto !important; }
验证：首页信任条读起来是完整一句话，不再是「208 reviews on」悬在半空。
P1 · #7约 30 分钟

拼写错误清理 + 社媒账号名统一

为什么：对一个以「把名字绣对」为卖点的品牌，站上的拼写错误是反向背书。
  1. 首页编辑，修 3 处文案（两张轮播 slide 都有）：treadsures → treasures；MADEWITH → MADE WITH
  2. 站内号召语写的是 @nameli_offical（official 拼错），而实际 Instagram 是 nameli38、TikTok 是 nanmeli0 —— 三个名字对不上，粉丝找不到人
  3. 建议：注册/改名一套统一 handle（如 nameli.official），IG、TikTok、站内文案、页脚链接四处对齐；改不了旧账号名就把站内文案改成真实 handle
验证：首页 Ctrl+F 搜 treadsures / MADEWITH 无结果；点页脚社媒图标能到达对应账号。
