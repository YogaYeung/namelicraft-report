严重级（先修，直接影响流量与转化）

严重 · #1 · SEO

首页 0 个 H1/H2 —— 标题被拆成 h5 单词碎片

Hero 大标题「Best Sellers」实际是两个独立的 <h5> 标签（其中一个为空），全页没有任何 H1/H2。Google 无法理解页面主题层级，配合 title 只有品牌词，自然流量入口等于焊死。
A 脚本实测：h1s = []（空数组）· 首页正文仅约 319 词 · title = "Nameli | Personalized Gifts"（无 baby/custom/embroidered 任一关键词）
修法：Hero 标题改 H1（含关键词，如 "Personalized Baby Shower Gifts & Name Baskets"），分区标题用 H2；title 重写为关键词+品牌结构。Blocksy 主题内即可改，不动功能。
首页全页 · 视觉丰富但搜索引擎读到的结构是空的
严重 · #2 · SEO/可访问

PDP 卖点全压在图片里 —— 文字描述仅 184 字符

主力产品 Baby Shower Basket（4.9★/108 评）的详情区：文字描述 184 字符，工艺/规格/故事全在 6 张长图里（2 张无 alt）。人眼看着丰富，Google/AI 搜索/读屏器一个字都读不到。全站 27/39 首页图片无 alt。
B 脚本实测（滚动懒加载后）：descLength=184 · descImgCount=6 · descImgNoAlt=2 · hasReviewSection=true
修法：每张卖点长图下补 200-300 词文字版（材质/尺寸/刺绣工艺/礼盒清单）+ 全部图片补描述性 alt。这是 SEO 命脉级修复。
PDP 首屏本身完整（评分/价格/色卡/定制框都在）
严重 · #3 · 移动转化

VIP OFFER 悬浮球 + 购物车按钮，移动端左右夹击盖住内容

移动 PDP 上，左下 VIP OFFER 悬浮球直接盖住产品副标题和卖点 bullets，右下购物车按钮盖住相邻内容；移动首页同样盖住产品卡标题。移动端是跨境流量大头，用户看卖点的黄金位置被自家挂件占了。
稳定复现：home + PDP、桌面 + 移动四处截图均在（非弹窗类状态依赖，无需双测豁免）
修法：VIP OFFER 移动端收成细条 banner 或延迟出现；两个悬浮件不同角错开；给内容区留 80px 底部安全距。
移动 PDP · VIP OFFER 压住副标题与卖点行
严重 · #4 · 可读性

「Write a Review」白字配奶茶底，对比度 1.71

评价区入口按钮白字 (#FFF) 配 rgb(220,194,165) 底，实测对比度 1.71（WCAG AA 要求 ≥4.5）——想留评的用户几乎看不清按钮。右侧色块为实色复刻：
Write a Review（实色复刻 1.71）
A 脚本实测：fg rgb(255,255,255) · bg rgb(220,194,165) · contrast=1.71 · 16px
修法：文字换深棕 #5C4327（同色系不破坏奶油调性，对比度 >7）或底色加深。
首访首屏：Klaviyo 弹窗 + cookie 条同屏（见下）

体验级（信任感漏气，低成本快修）

体验 · #5

信任条「208 reviews on ___」—— 平台 logo 渲染宽度 0，句子断半截

首页信任条本应显示「208 reviews on WiserReview」，但 logo 图（brand-icon-1-1.svg）实测渲染宽度为 0，桌面移动都只剩「208 reviews on」悬在半空——最重要的社会证明位自己漏气。
实测：wiserreview brand-icon getBoundingClientRect().width = 0（桌面+移动截图均可见断句）
修法：给 logo 容器设最小宽度/检查懒加载类名；或干脆换成自有文案「4.8/5 · 208 verified reviews」加星标。
体验 · #6

文案拼写错误 + 社媒 handle 全线混乱

首页两处 slide 均写 "treadsures"（应为 treasures）、"CUSTOM GIFT MADEWITH LOVE" 缺空格；页脚号召关注 @nameli_offical（official 拼错）而实际 IG handle 是 nameli38，TikTok 是 nanmeli0——三个名字对不上，对「把名字绣对」为卖点的品牌是反向背书。
首页 DOM 文本 + 页脚 href 实测：instagram.com/nameli38 · tiktok.com/@nanmeli0 · 文案写 @nameli_offical
修法：typo 当天可清；社媒 handle 建议统一注册/改名为 nameli.official 一套，全渠道对齐。
体验 · #7 · 仅首访（双测判定）

Klaviyo 10% off 弹窗落地 0 秒即弹 + cookie 条同屏叠加

首访落地瞬间：全屏 Klaviyo 邮箱弹窗（z-index 90000）+ 底部 cookie 条同时出现，用户没看到任何产品先吃两层打扰。复访双测不再弹（cookie 门控），故标「间歇/仅首访」不算致命——但首访恰恰是广告流量落地的那一次。
D 脚本双测：首访 fullscreenModalCount=2 → 复访=0 · 测试 locale zh-CN 已记录
修法：Klaviyo 触发改「滚动 50% 或停留 15s」；广告落地页流量尤其别 0 秒弹。
